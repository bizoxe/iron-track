**IronTrack** is an async backend research environment built with **FastAPI** and **Advanced-Alchemy**.

It is designed to explore architectural patterns, concurrency isolation, and performance trade-offs within a workout tracking domain.
The project focuses on experimentation and benchmarking rather than providing a production-ready solution.

---

Implemented Components
----------------------

I. Architecture and Design Patterns
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

* **Layered Architecture:** Strict separation of concerns: **`domain`** (logic, controllers, services), **`db`** (models and migrations), **`config`**, and **`lib`** (shared utilities).
* **Service and Repository Patterns:** Implemented via **Advanced-Alchemy** for asynchronous **SQLAlchemy** operations and isolation of business logic.
* **Schema Evolution:** Uses **Alembic** for versioned database schema migrations.

II. Performance & Optimization
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

* **ASGI Runtime:** ASGI runtime powered by **Granian**.
* **Serialization:** Implements **MsgSpecJSONResponse** using **msgspec** to reduce serialization overhead.
* **Connection Management:** **PgBouncer** integration for transaction pooling.
* **Observability:** Custom ASGI middleware for request timing and metric logging.

III. Security & Administration
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

* **Session Management:** JWT Access/Refresh tokens via secure HTTP-only cookies, featuring user data caching (**cashews**) and token blacklisting.
* **Access Control (RBAC):** CRUD operations for accounts with Role-Based Access Control supporting `superuser` and `trainer` roles.
* **Command Line Interface:** Centralized entry point built on **Typer** for server management, migrations, and administrative tools.

IV. Engineering Practices & CI/CD
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

* **Edge Proxy:** Configured with **Angie** as a reverse proxy for TLS termination.
* **Containerization:** Multi-stage **Docker** builds (Builder/Runner) with dependency optimization via **uv**.
* **Local deployment workflow:** Orchestrated via `docker-compose` with automated migrations and service health checks.
* **Packaging:** Built and packaged using **Hatchling**.
* **Testing & Standards:**
    * **Automated Tests:** **Pytest** environment with transactional rollbacks for isolated integration testing.
    * **Static Analysis:** **Ruff** (linting/formatting) and **Mypy** (type checking) with **pre-commit** integration.
    * **Logging:** Structured JSON logging via **structlog**.

V. Data & Content
~~~~~~~~~~~~~~~~~

* **Data Seeding:** Includes an automated script (`seeder.py`) to populate the database with **1000 ready-to-use exercises**, along with muscle group and equipment directories.
